## 🧠 About Me

I like working close to systems, backend, and anything where performance actually matters.

I'm the creator of **Lu Language**, a custom programming language and interpreter built from the ground up in pure C, complete with a standalone **stack-based virtual machine**:

```
source.lu
   │
   ▼
┌─────────┐    ┌────────┐    ┌─────┐    ┌──────────┐    ┌──────────┐    ┌─────┐    ┌───────────────────┐
│  Lexer  │ →  │ Parser │ →  │ AST │ →  │ Compiler │ →  │ Bytecode │ →  │ VM  │ →  │ Runtime Execution │
└─────────┘    └────────┘    └─────┘    └──────────┘    └──────────┘    └─────┘    └───────────────────┘
```

- 🔭 Currently building **Lu Language** and its standalone stack-based VM
- 🛠️ Next project: building a **shell** from scratch
- 🌱 Deep in the weeds of interpreters, bytecode design, and VM internals
- 💬 Ask me about lexers, parsers, ASTs, or how a stack machine actually executes your code
- ⚡ I'm into systems programming, backend, and squeezing out performance
